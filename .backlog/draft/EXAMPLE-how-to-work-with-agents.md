# Пример: Как работать с агентами

## Шаг 1: Создайте файл задачи здесь (в draft/)

```markdown
# Task: Спроектировать архитектуру платёжной системы

## Описание
Нужна архитектура для интеграции онлайн-платежей в систему бронирования.

## Требования
- Интеграция со Stripe
- Поддержка рекуррентных платежей
- Firebase Functions для webhook'ов
- Безопасное хранение платёжной информации

## Ожидаемый результат
- Архитектурный документ
- OpenAPI спецификация для платёжного API
- Диаграмма последовательности операций
- ADR (Architecture Decision Record) по выбору платёжного провайдера
```

## Шаг 2: Когда задача готова - переместите в pending/

```bash
# Переименуйте и переместите файл
mv .backlog/draft/my-task.md .backlog/pending/feature-050-payment-architecture.md
```

## Шаг 3: Попросите Claude обработать бэклог

```
Обработай задачи из .backlog/pending/
```

---

## Альтернатива: Прямой запрос

Вместо создания файла задачи, просто попросите:

```
Спроектируй архитектуру платёжной системы со Stripe для zabicekiosk.
Нужна поддержка рекуррентных платежей и webhook'ов.
```

Claude автоматически делегирует задачу lean-architect агенту.

---

**Примечание**: Файлы в draft/ НЕ обрабатываются автоматически.
Это ваше личное пространство для заметок и черновиков.
